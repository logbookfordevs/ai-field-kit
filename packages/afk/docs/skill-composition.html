Think in components, then ship skills.

React made reuse feel physical: tiny parts, named compositions, and clear props. AFK borrows that instinct for agent behavior.

The React-shaped thought

A wrapper is the component people reach for. Primitives are the children that make the behavior reusable.

The AFK-shaped catalog

The catalog says the same thing without pretending it is runtime code.

    "codebase-design"

component role names the shape.
children composes names the dependency tree.
props autoInvocation declares discovery behavior.
app route A flow assembles skills into a larger run.

Six roles. One decision.

Each skill should reveal whether it is a reusable part, a named experience, or a larger operating mode.

primitive

Reusable behavior

Small enough to compose. Useful on its own. Good candidate for model discovery.

wrapper

Named experience

A user-facing skill that bundles primitives into a focused workflow.

flow

Repeatable motion

A larger run shape that coordinates phases, packets, tracking, or execution.

utility

Specific helper

Direct operational support for pickup, handoff, session work, or terminal coordination.

reference

Knowledge surface

A preference map, registry guide, or lookup surface that informs another action.

router

Skill selection

A compass that chooses the smallest useful next skill without becoming the workflow.

autoInvocation says the real thing.

The field is about discovery, not permission. A discoverable skill can still be invoked manually.

Not "model vs user"

A model-discoverable skill can still be invoked by a person. A manual skill is not forbidden to the model forever. It just stays out of automatic discovery until someone asks for it.

autoInvocation: trueExpose this skill to automatic model discovery.
autoInvocation: falseKeep this as a manually invoked wrapper, flow, or specialist surface.

The usual AFK instinct

Primitives are often model-discoverable. Wrappers and flows are usually manual. References depend on how broad they are.

primitive
Small contextGood to discover automatically when the topic appears.
wrapper
Named intentBetter when a person asks for the experience directly.
flow
High agencyUsually too much motion to trigger from a casual phrase.
reference
Judgment callAuto when broadly helpful, manual when situational.

Wrappers are the UX. Primitives are the leverage.

The person asks for one experience. The skill can still be built from smaller pieces.

A skill entry should explain itself.

The catalog becomes a system map: shape, discovery, composition, future profile membership.

Four questions

A well-shaped entry should answer these quickly.

01
What is it?`role` gives the compositional shape.
02
Can it auto-discover?`autoInvocation` controls model discovery.
03
What does it build on?`composes` names the dependency story.
04
Where will it live?`profiles` prepares future activation sets.

The CLI can keep selection simple while still showing the composed children before install.

Select wrapperThe main list stays flat so every skill remains a real selectable item.
step 02
Review composed skillsAFK shows the primitives the wrapper is built from in a second prompt.
step 03
Install the setThe user can keep or remove suggested children before installation.
step 04
Sync invocationAFK writes policy metadata so auto-discovery follows the catalog.
