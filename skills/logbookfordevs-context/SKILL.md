# LogbookForDevs Skill

## Purpose
Provide context and guidelines for tasks related to LogbookForDevs, a public hub of side projects, experiments, and open-source tools for developers. This skill helps ensure consistency of voice, structure, and purpose when generating or modifying content tied to this project.

## Guidelines
- LogbookForDevs is a technical logbook emphasizing practical building, documentation, and sharing of developer-focused tools and experiments.
- Prioritize **clarity, maintainability, and usefulness** of outputs.
- Avoid unnecessary complexity; prefer well-scoped, independently useful artifacts.
- Documentation and explanations should be **accessible and practical** for developers.
- Tone should be technical, direct, and focused on problem → rationale → solution → learnings.
- When generating code, include minimal reproducible examples and lightweight tests where applicable.

## When to Use
Reference this skill when:
- Generating or refining content for LogbookForDevs (readmes, articles, docs)
- Explaining architecture, patterns, or code linked to LogbookForDevs
- Creating formatted outputs intended for publication under LogbookForDevs
- Designing examples, tutorials, or demos in the context of the project

## Language
- Draft content can be in Portuguese, but final outputs for public publication should default to **English** unless otherwise specified.
