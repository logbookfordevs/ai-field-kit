# Logbook for Devs Context

## Purpose
Define mandatory standards for any project that uses the Logbook for Devs brand, website, or identity.
This skill exists to keep brand usage, attribution, and support UX consistent across all derivatives.

## Scope (Mandatory)
Use this skill whenever a project:
- Uses the **Logbook for Devs** name, brand, or identity.
- References the official website `https://logbookfordevs.com/`.
- Is a derivative, subproject, experiment, or tool published under the brand.

If any of the above is true, this skill is **required**.

## Core Guidelines
- LogbookForDevs is a technical logbook emphasizing practical building, documentation, and sharing of developer-focused tools and experiments.
- Prioritize **clarity, maintainability, and usefulness** of outputs.
- Avoid unnecessary complexity; prefer well-scoped, independently useful artifacts.
- Documentation and explanations should be **accessible and practical** for developers.
- Tone should be technical, direct, and focused on problem → rationale → solution → learnings.
- When generating code, include minimal reproducible examples and lightweight tests where applicable.

## Brand Attribution (Mandatory)
Every branded/derivative project must include this attribution block somewhere in the UI or docs (usually near the end, commonly in the footer, but placement is flexible by design):

- `A tool from the [Logbook for Devs](https://logbookfordevs.com/)`
- `Charting the technical seas, one commit at a time.`

Rules:
- Keep the destination URL exactly: `https://logbookfordevs.com/`
- Keep the second sentence exactly as written.
- Visual placement can vary to match each project's layout and narrative.

## When to Use
Reference this skill when:
- Generating or refining content for LogbookForDevs (readmes, articles, docs)
- Explaining architecture, patterns, or code linked to LogbookForDevs
- Creating formatted outputs intended for publication under LogbookForDevs
- Designing examples, tutorials, or demos in the context of the project
- Building or maintaining the LogbookForDevs website or projects related to the brand

## Language
- Write outputs in **English** unless the user explicitly requests another language.

## Donation & Support (Mandatory)
All branded/derivative projects must include:
1. Ko-fi preset thematic options
2. A Ko-fi button without preset amount
3. A Buy Me a Coffee button without preset amount

Required links:
- Ko-fi base: `https://ko-fi.com/logbookfordevs`
- Buy Me a Coffee base: `https://buymeacoffee.com/logbookfordevs`

Ko-fi preset options:
- Buy me a coffee (5): `https://ko-fi.com/logbookfordevs?amount=5`
- Buy me lunch (15): `https://ko-fi.com/logbookfordevs?amount=15`
- Buy me dinner (30): `https://ko-fi.com/logbookfordevs?amount=30`

Rules:
- Never include only one platform; always show **both** Ko-fi and Buy Me a Coffee.
- Keep support CTAs integrated naturally into the project's narrative/theme.
- Adapt copy, styling, and layout to the design context, while preserving the required links and option mapping.

## Compliance Checklist
Before finalizing a branded/derivative project, confirm all items:
- Attribution line includes `A tool from the [Logbook for Devs](https://logbookfordevs.com/)`
- Tagline includes `Charting the technical seas, one commit at a time.`
- Ko-fi presets for 5, 15, and 30 are present
- Ko-fi base button is present
- Buy Me a Coffee base button is present
