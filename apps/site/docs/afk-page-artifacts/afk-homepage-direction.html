Why It Exists

Agent Habits Fragment Fast.

Every coding agent has its own memory surface, config shape, tool wiring, and idea of “how this project works.” AFK gives those surfaces a shared operating layer without becoming another agent.

Problem

Standards Drift By Tool.

Codex may know one review doctrine, Cursor another, and a terminal agent something else entirely. The cost is not one bad answer; it is maintaining the same habits repeatedly.

Pattern

Context Belongs Near The Work.

Project-specific rules, setup choices, hooks, and manifests should travel with the repository when they matter, while personal defaults stay reusable across machines.

Position

AFK Is The Field Kit, Not The Agent.

It prepares the standards and utilities your local agents inherit. You keep choosing the agent surface; AFK keeps the working system coherent.

Layer 01 Rules

Always-on standards for how agents should work in a repo.

Layer 02 Skills

Reusable playbooks for reviews, docs, debugging, planning, and execution.

Layer 03 Runtime Utilities

Hooks, MCPs, setup flows, and small helpers that make the kit operational.

What Ships In The Kit

Field Manual, Not Card Wall.

rules

Shared working standards.

Capture repository doctrine, agent behavior, safety boundaries, and review expectations in versioned files.

skills

Reusable expert routines.

Install focused instructions for common work: documentation, debugging, task breakdown, tradeoff analysis, reviews, and handoff.

workflows

Repeatable multi-step operating modes.

Use lightweight process when the task needs discovery, planning, checkpoints, validation, or a durable artifact.

hooks

Guardrails at the moment of action.

Prompt, stop, or route work when an agent is about to cross a project boundary or miss a required verification step.

MCPs

Tool wiring that moves with the setup.

Keep external capabilities discoverable through the same field kit instead of rebuilding tool context per agent.

utilities

Small commands for local operating hygiene.

Support installation, syncing, diagnostics, and repo-local setup without turning the homepage into a docs manual.

Two Setup Paths

Choose The Amount Of Kit You Want.

Some developers only want portable skills. Others want the CLI, manifests, hooks, MCPs, and project-local conventions working together.

Path 01

Get Skills Only.

Best when you want the reusable AFK skill library inside an existing agent setup without adopting the full CLI stack yet.

Command npx skills add https://github.com/logbookfordevs/ai-field-kit
  • Smallest install path for skill discovery and reuse.
  • Does not try to own your agent stack or project manifests.
  • Useful when testing AFK one workflow at a time.

Path 02

Install The AFK CLI Stack.

Best when you want rules, skills, workflows, hooks, MCPs, and utilities managed as one operating layer across local agents.

Command curl -fsSL https://ai-field-kit.logbookfordevs.com/install.sh | bash
  • Sets up the portable field kit rather than one isolated skill.
  • Designed for multi-agent developers who move between surfaces.
  • Leaves production command details for repo-verified docs.
